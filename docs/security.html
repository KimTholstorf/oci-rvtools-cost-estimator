Security & Privacy

OCI RVTools Cost Estimator — how your data is handled

Your files never leave your browser

File upload stays local

Your RVTools .xlsx file is read by your browser using the standard File API. It is loaded into a temporary in-memory space inside the browser tab — it is never transmitted over the network to any server.

Python runs inside your browser

The estimator logic runs using Pyodide — a full Python environment compiled to WebAssembly that executes entirely within your browser tab. There is no backend server receiving or processing your data.

No server — just static files

This tool is hosted on GitHub Pages as a static website. There is no application server, no database, and no API endpoint that could accept or store uploaded files. GitHub Pages is physically incapable of receiving file uploads.

Only one outbound request

The only data fetched from the internet is prices.json — a read-only file containing cached Oracle Cloud list prices. No part of your RVTools data is included in any network request.

Output is downloaded directly

The generated cost estimate Excel file is assembled in browser memory and handed directly to your browser as a download. It is never uploaded or stored anywhere outside your device.

Data is discarded when you close the tab

All data — your uploaded file, intermediate processing, and the output — lives only in the WebAssembly memory allocated to the browser tab. Closing or refreshing the tab wipes everything. Nothing is written to localStorage, cookies, or any persistent storage.

100% client-side — zero data exposure

Verify it yourself

Open your browser's DevTools (F12 or ⌘⌥I), go to the Network tab, then run the estimator. You will see requests only for pyodide.js, core.py, prices.json, and Pyodide package files. No request will contain your RVTools file data.
